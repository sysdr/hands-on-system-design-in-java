# TwitterClone – Durable Tweet Storage (WAL)

This directory is the **standalone project** (push this to git). It does not depend on the parent `setup.sh`.

## Requirements

- **JDK 17+**: `javac` and `java` in PATH, or `JAVA_HOME` set, or a local JDK in `.jdk/` or `../.jdk/`
- **netcat** (`nc`): for client tests (often pre-installed)
- **Docker** (optional): for containerized run

## Quick start (independent of setup.sh)

1. **Build** (from this directory):
   ```bash
   cd /path/to/TwitterClone
   ./build.sh
   ```

2. **Start server:**
   ```bash
   ./start-server.sh
   ```

3. **Run tests** (server must be running, in another terminal):
   ```bash
   ./run-tests.sh
   ```

4. **Run demo** (updates dashboard metrics):
   ```bash
   ./run-demo.sh
   ```

5. **View dashboard:**
   ```bash
   ./view-dashboard.sh
   ```

6. **Cleanup** (stop server, Docker, prune):
   ```bash
   ./cleanup.sh
   ```

## Layout

- `src/main/java/...` – Java source (generated by parent `setup.sh` if present; otherwise ensure sources exist before `./build.sh`)
- `target/TwitterClone.jar` – built by `./build.sh`
- `data/wal/`, `data/dashboard_metrics.json` – runtime data
- `dashboard.html`, `dashboard_out.html` – dashboard template and output

## If using parent setup.sh

From the **parent** directory (lesson2), running `./setup.sh` will generate source into this project, build, run tests, and update dashboard metrics. Only this **TwitterClone** directory needs to be pushed to git.
